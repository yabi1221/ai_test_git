# IW 그룹 프로젝트별 사용 기술 정리

이 문서는 `iw_group_analysis`에 정리된 기존 분석 결과를 바탕으로, 각 프로젝트에서 사용된 프로그래밍 기술과 운영 기술을 학습용으로 한 번에 모아둔 요약본이다.

## 1. iop-iw-api

### 주요 언어
- Java 8

### 핵심 프레임워크
- Spring Boot 2.7.18
- Spring Security
- Spring Batch
- Spring Scheduling
- Spring Async
- MyBatis
- 일부 JPA 의존성

### 인증/보안
- JWT
- Keycloak 연동
- API Key 인증 필터

### 데이터/메시징/스토리지
- MariaDB
- PostgreSQL
- Kafka
- AWS S3

### 문서/도구
- Swagger
- springdoc

### 빌드
- Maven
- Gradle

### 성격
- 핵심 업무 백엔드
- 대형 모놀리식 Spring API

## 2. iop-iw-ui

### 주요 언어
- TypeScript
- JavaScript

### 핵심 프레임워크/라이브러리
- React
- Vite
- React Router
- React Query
- Axios
- TailwindCSS

### 인증/클라이언트 연동
- Keycloak 클라이언트 연동
- Bearer Token 처리
- Refresh Token 흐름

### 프론트엔드 구조 특징
- SPA 구조
- Module Federation 흔적 존재
- 외부 원격 프론트 모듈 연동

### 빌드/배포
- Node.js 기반 빌드
- Nginx
- Docker

### 성격
- 포털 셸 역할의 프론트엔드
- API, 업로드 서버, 외부 원격 앱을 묶는 사용자 진입점

## 3. iop-iw-tus

### 핵심 기술
- tusd
- TUS Protocol

### 운영/배포 기술
- Docker
- Dockerfile
- DockerfileK8s
- Jenkins

### 연동 기술
- HTTP Hook
- 업로드 후처리 콜백

### 성격
- 대용량 파일 업로드 전용 서버
- 업무 로직보다 업로드 인프라에 가까운 컴포넌트

## 4. iop-iw-gpki

### 주요 언어/구성
- Java
- JSP

### 핵심 프레임워크/플랫폼
- eGovFrame Boot Parent 4.3.0
- Servlet 기반 WAR 애플리케이션
- Tomcat 배포 구조

### 보안/인증 기술
- GPKI
- gpkiapi
- gpkisecureweb
- gpkisecurewebkeyboard

### 빌드/배포
- Maven
- WAR 패키징
- Docker

### 성격
- GPKI 로그인 화면과 결과 처리를 담당하는 특수 웹앱
- Keycloak GPKI 인증 흐름의 보조 컴포넌트 가능성 높음

## 5. iop-iw-manifest

### 핵심 기술
- Helm
- ArgoCD
- Kubernetes 배포 values 관리

### 구성 방식
- 환경별 values 분리
- dev / prd 분리
- 서비스별 chart 분리

### 관리 대상
- UI
- API
- TUS
- GPKI
- Scheduler

### 성격
- 애플리케이션 코드 저장소가 아니라 배포/운영 정의 저장소
- 실제 서비스 연결 관계와 라우팅을 가장 명확히 보여주는 저장소

## 6. keycloak

### 주요 언어
- Java 11

### 핵심 플랫폼
- Keycloak 26
- Keycloak SPI

### 구현 기술
- Custom User Storage Provider
- Custom Authenticator
- Event Listener Provider
- JDBC 사용자 저장소 연동

### 확인된 인증 확장
- GPKI Authenticator
- MetaMask Authenticator
- Session Limit Event Listener

### 빌드
- Maven
- Maven Shade Plugin

### 성격
- Keycloak 본체가 아니라 Keycloak 확장 Provider 저장소
- 포털 인증 구조의 핵심 확장 코드

## 7. user-storage-jpa-master

### 주요 언어
- Java 17

### 핵심 플랫폼
- Keycloak 26.0.7
- Keycloak Provider 확장 구조

### 구현 기술
- JDBC User Storage Provider
- Improved User Storage Provider
- GPKI Authenticator
- Improved GPKI Authenticator
- Mobile QR Authenticator
- MetaMask Authenticator
- Session Limit Event Listener

### 데이터/실행 환경
- MariaDB
- Docker
- Docker Compose

### 빌드
- Maven

### 성격
- Keycloak 사용자 저장소/인증 확장 실험 또는 이전 버전 계열 저장소
- `keycloak` 프로젝트와 역할이 상당히 겹침

## 학습 우선순위 제안

이 그룹을 이해하려면 아래 순서가 가장 효율적이다.

1. `iop-iw-ui`: 사용자 화면과 호출 시작점
2. `iop-iw-api`: 핵심 업무 로직과 데이터 처리
3. `keycloak`: 로그인과 사용자 저장소 확장
4. `iop-iw-tus`: 파일 업로드 흐름
5. `iop-iw-manifest`: 실제 배포 연결 구조
6. `iop-iw-gpki`: 특수 인증 보조 웹앱
7. `user-storage-jpa-master`: Keycloak 확장 비교/참고 저장소
