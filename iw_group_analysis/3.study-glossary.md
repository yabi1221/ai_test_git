# IW 그룹 용어집

이 문서는 IW 그룹 프로젝트를 읽을 때 자주 나오는 기술 용어를, 이 프로젝트 맥락에 맞춰 이해하기 쉽게 정리한 용어집이다.

## API

Application Programming Interface의 줄임말이다.  
이 그룹에서는 주로 `iop-iw-api`가 제공하는 HTTP 업무 기능을 뜻한다.

예:
- 문서 조회
- 첨부파일 메타데이터 처리
- 사용자/권한 조회
- 관리자 기능

## REST API

HTTP 기반으로 데이터를 주고받는 API 스타일이다.  
이 그룹에서는 UI가 `iop-iw-api`를 호출할 때 거의 이 방식으로 동작한다고 보면 된다.

## SPA

Single Page Application의 줄임말이다.  
브라우저에서 페이지 전체를 새로고침하지 않고 화면 일부만 바꾸는 프론트엔드 방식이다.

이 그룹에서는 `iop-iw-ui`가 SPA 역할을 한다.

## React

컴포넌트 단위로 UI를 만드는 프론트엔드 라이브러리다.  
이 그룹에서는 `iop-iw-ui`의 화면 구성을 담당한다.

## TypeScript

JavaScript에 타입 시스템을 추가한 언어다.  
이 그룹에서는 `iop-iw-ui`의 주요 개발 언어다.

## Vite

프론트엔드 개발 서버와 빌드 도구다.  
이 그룹에서는 `iop-iw-ui`를 개발/빌드할 때 사용된다.

## Axios

브라우저나 Node.js에서 HTTP 요청을 보내는 라이브러리다.  
이 그룹에서는 UI가 API 서버에 요청을 보낼 때 사용된다.

## React Query

서버 데이터를 가져오고 캐싱하는 데 쓰는 라이브러리다.  
이 그룹에서는 API 호출 결과를 화면 상태와 연계하는 데 사용된다.

## TailwindCSS

유틸리티 클래스 기반 CSS 프레임워크다.  
이 그룹에서는 `iop-iw-ui`의 스타일 구성에 사용된다.

## Spring Boot

Java 기반 백엔드 애플리케이션을 빠르게 구성하는 프레임워크다.  
이 그룹에서는 `iop-iw-api`의 핵심 기반 기술이다.

## Controller

HTTP 요청을 직접 받는 백엔드 진입점 클래스다.  
이 그룹에서는 URL 요청이 먼저 Controller에 들어오고, 이후 Service로 전달된다.

## Service

업무 로직을 담당하는 계층이다.  
이 그룹에서는 문서 처리, 첨부 처리, 인증 처리 같은 핵심 로직이 Service에 많이 들어 있다.

## Mapper

MyBatis에서 Java 메서드와 SQL을 연결하는 계층이다.  
이 그룹에서는 `iop-iw-api`가 MyBatis 중심 구조라 Mapper 역할이 중요하다.

## MyBatis

Java 코드와 SQL을 매핑하는 persistence 기술이다.  
JPA처럼 자동 생성에 크게 의존하기보다, SQL을 직접 관리하는 데 강하다.

이 그룹에서는 `iop-iw-api`의 주요 DB 처리 방식이다.

## JPA

Java ORM 표준이다.  
이 그룹에서는 `iop-iw-api`에 일부 의존성이 보이지만, 중심은 MyBatis다.

## JWT

JSON Web Token의 줄임말이다.  
로그인 후 사용자 인증 상태를 담아 전달하는 토큰 형식이다.

이 그룹에서는 UI가 토큰을 들고 API를 호출하는 흐름에서 중요하다.

## Access Token

실제 API 호출 시 사용하는 토큰이다.  
이 그룹에서는 UI가 API 요청에 Bearer Token으로 붙여 보낸다.

## Refresh Token

만료된 Access Token을 다시 발급받기 위한 토큰이다.  
이 그룹에서는 UI가 로그인 상태를 유지하는 데 사용된다.

## Bearer Token

HTTP Authorization 헤더에 붙여 보내는 토큰 방식이다.  
보통 `Authorization: Bearer ...` 형태로 전송된다.

## Spring Security

Spring 기반 인증/인가 프레임워크다.  
이 그룹에서는 `iop-iw-api`가 요청 보호, 필터 체인, 인증 검증을 처리할 때 사용한다.

## Authentication

사용자가 누구인지 확인하는 과정이다.  
이 그룹에서는 로그인과 토큰 검증이 여기에 해당한다.

## Authorization

인증된 사용자가 무엇을 할 수 있는지 결정하는 과정이다.  
이 그룹에서는 권한에 따라 메뉴, 기능, API 접근이 달라질 수 있다.

## Keycloak

오픈소스 IAM(Identity and Access Management) 솔루션이다.  
이 그룹에서는 로그인과 사용자 인증의 중심이다.

단, 워크스페이스의 `keycloak` 프로젝트는 Keycloak 본체가 아니라 확장 Provider 코드 저장소다.

## Realm

Keycloak에서 사용자, 클라이언트, 인증 흐름을 묶는 보안 영역이다.  
이 그룹에서는 UI 환경변수에 realm 값이 들어가며, 로그인 대상을 구분한다.

## Client

Keycloak이 인증해주는 애플리케이션 단위다.  
이 그룹에서는 `iop-iw-ui`가 대표적인 client다.

## OIDC

OpenID Connect의 줄임말이다.  
OAuth2 위에 사용자 인증 정보를 얹은 표준이다.

이 그룹의 Keycloak 로그인 흐름도 이 개념으로 이해하면 된다.

## OAuth2

권한 위임을 위한 표준 프로토콜이다.  
이 그룹에서는 로그인, 토큰 발급, 토큰 갱신 흐름을 이해할 때 필요하다.

## Keycloak SPI

Service Provider Interface의 줄임말이다.  
Keycloak 기능을 확장하기 위한 개발 방식이다.

이 그룹에서는 아래 기능들이 SPI로 구현돼 있다.
- JDBC 사용자 저장소
- GPKI 인증기
- MetaMask 인증기
- 세션 제한 이벤트

## User Storage Provider

Keycloak이 자체 사용자 저장소 대신 외부 DB 사용자 정보를 읽도록 확장하는 기능이다.  
이 그룹에서는 JDBC 기반으로 업무 DB 사용자 정보를 참조할 가능성이 높다.

## Authenticator

Keycloak 로그인 흐름 중 특정 인증 단계를 직접 구현하는 확장 컴포넌트다.  
이 그룹에서는 `GpkiAuthenticator`, `MetaMaskAuthenticator` 등이 여기에 해당한다.

## Event Listener

Keycloak 내부 이벤트를 감지해 정책을 수행하는 확장 기능이다.  
이 그룹에서는 세션 수 제한 같은 운영 정책에 사용된다.

## GPKI

Government Public Key Infrastructure의 줄임말이다.  
공공 인증서 기반 로그인/보안 체계라고 이해하면 된다.

이 그룹에서는 `keycloak`의 GPKI 인증기와 `iop-iw-gpki` 웹앱이 함께 관련된다.

## MetaMask

블록체인 지갑 기반 인증에 쓰일 수 있는 확장 포인트다.  
이 그룹의 Keycloak 확장 안에는 관련 인증기 클래스가 존재한다.

## QR 인증

QR 코드를 이용한 로그인/인증 방식이다.  
이 그룹에서는 `user-storage-jpa-master`에 `MobileQrAuthenticator` 흔적이 보인다.

## TUS

대용량 파일을 끊김 없이 이어서 올릴 수 있게 해주는 업로드 프로토콜이다.  
이 그룹에서는 `iop-iw-tus`가 이 역할을 맡는다.

## tusd

TUS 서버 구현체다.  
이 그룹에서는 `iop-iw-tus`가 별도 애플리케이션이 아니라 `tusd` 중심으로 구성돼 있다.

## Hook

어떤 이벤트가 발생했을 때 다른 시스템에 알려주는 콜백 방식이다.  
이 그룹에서는 TUS 업로드 완료 후 API `/internal/tusd`로 호출하는 흐름이 대표적이다.

## Callback

특정 작업이 끝난 뒤 다른 URL이나 함수로 다시 알려주는 방식이다.  
이 그룹에서는 로그인 결과 처리나 업로드 후처리 흐름에서 자주 등장한다.

## MariaDB

관계형 데이터베이스다.  
이 그룹에서는 `iop-iw-api`의 메인 업무 DB이자, Keycloak 사용자 저장소 후보 DB로 보인다.

## PostgreSQL

관계형 데이터베이스다.  
이 그룹에서는 공통 사용자/조직 동기화 계열 DB로 쓰일 가능성이 있다.

## DataSource

애플리케이션이 DB 연결을 관리하는 설정 단위다.  
이 그룹에서는 `iop-iw-api`가 MariaDB와 PostgreSQL을 나눠 쓰기 때문에 중요하다.

## Transaction

DB 작업을 하나의 묶음으로 처리하는 개념이다.  
중간에 실패하면 롤백하고, 모두 성공하면 커밋한다.

## Scheduler

정해진 시간이나 주기에 따라 작업을 자동 실행하는 기능이다.  
이 그룹에서는 API 또는 별도 `iop-iw-sch`가 관련될 가능성이 있다.

## Batch

대량 데이터 처리나 정기 실행 작업에 쓰는 방식이다.  
이 그룹의 `iop-iw-api`에는 Spring Batch 흔적이 있다.

## Kafka

메시지 브로커/이벤트 스트리밍 플랫폼이다.  
이 그룹에서는 백엔드 외부 연동 축 중 하나다.

## S3

오브젝트 스토리지다.  
이 그룹에서는 첨부 또는 문서 파일 저장과 연계될 가능성이 있다.

## RAG API

Retrieval-Augmented Generation 관련 외부 AI API를 뜻한다.  
이 그룹에서는 `iop-iw-api`가 외부 RAG 시스템과 연동하는 흔적이 있다.

## Nginx

웹 서버이자 Reverse Proxy다.  
이 그룹에서는 `iop-iw-ui` 배포 구조에서 프론트 서빙과 프록시 역할에 사용된다.

## Reverse Proxy

클라이언트 요청을 받아 뒤쪽 서비스로 전달해주는 서버 역할이다.  
이 그룹에서는 `/iw`, `/iw/api`, `/iw/files` 같은 경로 분기에 중요하다.

## Docker

애플리케이션을 컨테이너로 패키징하는 기술이다.  
이 그룹 대부분의 프로젝트가 Docker 기반 배포 흔적을 가진다.

## Docker Compose

여러 컨테이너를 한 번에 실행하는 도구다.  
이 그룹에서는 `user-storage-jpa-master`의 로컬 실행/테스트 구성에서 보인다.

## WAR

Java 웹 애플리케이션 배포 패키지 형식이다.  
이 그룹에서는 `iop-iw-gpki`가 WAR로 패키징된다.

## JSP

Java 기반 서버 사이드 뷰 기술이다.  
이 그룹에서는 `iop-iw-gpki`의 로그인/결과 화면에 사용된다.

## Tomcat

Java 웹 애플리케이션 서버다.  
이 그룹에서는 `iop-iw-gpki` WAR를 올리는 배포 대상으로 쓰인다.

## Helm

Kubernetes 애플리케이션 배포 템플릿 도구다.  
이 그룹에서는 `iop-iw-manifest`가 서비스별 chart를 관리할 때 사용한다.

## ArgoCD

GitOps 방식으로 Kubernetes 배포를 관리하는 도구다.  
이 그룹에서는 manifest 저장소의 application YAML들이 여기에 해당한다.

## Kubernetes

컨테이너 오케스트레이션 플랫폼이다.  
이 그룹은 manifest 구조상 Kubernetes 배포를 기준으로 운영되는 것으로 보인다.

## values 파일

Helm 차트에서 환경별 설정값을 담는 파일이다.  
이 그룹에서는 dev/prd 차이를 읽을 때 가장 중요하다.

## Ingress / 라우팅

외부 요청을 어떤 서비스로 보낼지 정하는 설정이다.  
이 그룹에서는 `/iw`, `/iw/api`, `/iw/files` 분기가 핵심이다.

## Module Federation

여러 프론트엔드 앱을 런타임에 조합하는 방식이다.  
이 그룹에서는 `iop-iw-ui`가 외부 위젯/AI/카탈로그 모듈을 붙일 때 관련 흔적이 있다.

## 환경변수

실행 시 외부에서 주입하는 설정값이다.  
이 그룹에서는 API 주소, Keycloak 주소, 업로드 주소, 외부 원격 앱 주소를 주입하는 데 사용된다.

## 모놀리식 구조

여러 업무 기능이 하나의 큰 애플리케이션 안에 같이 들어 있는 구조다.  
이 그룹의 `iop-iw-api`는 대형 모놀리식 백엔드에 가깝다.

## 한 줄 요약

이 그룹을 이해할 때 가장 자주 쓰는 핵심 용어는 아래다.

- 화면: React, Vite, SPA
- 업무 API: Spring Boot, MyBatis, JWT
- 인증: Keycloak, OIDC, SPI, User Storage Provider, Authenticator
- 업로드: TUS, tusd, Hook
- 운영: Docker, Helm, ArgoCD, Kubernetes
