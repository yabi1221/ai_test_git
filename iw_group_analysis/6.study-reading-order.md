# IW 그룹 실전 읽기 순서

이 문서는 IW 그룹을 처음 분석하는 사람이 실제 파일을 어떤 순서로 열어야 하는지 정리한 실전용 가이드다.  
목표는 "전체 구조를 빠르게 잡는 것"이지, 처음부터 모든 코드를 정독하는 것이 아니다.

## 1. 먼저 알아둘 원칙

- 처음에는 모든 프로젝트를 깊게 읽지 않는다.
- 항상 `UI -> API -> Keycloak -> TUS -> Manifest` 순서로 본다.
- 한 파일을 자세히 파기보다, 여러 파일을 얕게 연결해서 흐름을 잡는다.
- 막히면 "이 파일은 누구와 연결되는가?"만 먼저 확인한다.

## 3. 첫 20분: UI부터 본다

UI는 사용자의 시작점이기 때문에 가장 먼저 보는 것이 효율적이다.

### 우선 열 파일

1. `iop-iw-ui/.env`
2. `iop-iw-ui/.env.production`
3. `iop-iw-ui/vite.config.ts`
4. `iop-iw-ui/src/config/api.ts`

### 여기서 볼 것

- API 주소가 어디인지
- 업로드 주소가 어디인지
- Keycloak 주소/realm/client가 무엇인지
- `/iw` 기준으로 서비스되는지

### 다음으로 볼 파일

1. 인증 관련 service 파일
2. 로그인 진입점 컴포넌트
3. 업로드 관련 service 또는 hook 파일

### 이 단계가 끝나면 답할 수 있어야 하는 질문

- UI는 어떤 서버들과 직접 통신하는가?
- 로그인은 어디로 보내는가?
- API 호출은 어떤 공통 설정을 쓰는가?
- 파일 업로드는 어디로 보내는가?

## 4. 다음 20분: API를 본다

4. 연결된 Mapper / XML

### 이 단계가 끝나면 답할 수 있어야 하는 질문

- API는 어떤 기술 스택으로 구성되는가?

## 9. 두 번째 라운드 읽기 순서

첫 1시간이 끝난 뒤에는 아래 순서로 조금 더 깊게 들어간다.

1. UI에서 실제 많이 쓰는 화면 하나를 고른다.
2. 그 화면이 호출하는 API를 찾는다.
3. API Controller -> Service -> Mapper -> XML을 따라간다.
4. 인증이 걸리면 Security / Keycloak 검증 코드를 본다.
5. 업로드가 있으면 TUS와 `/internal/tusd`를 같이 본다.
6. 운영 반영 구조는 Manifest로 확인한다.

## 10. 추천 실전 추적 과제

처음 읽을 때 가장 좋은 과제는 아래 셋이다.

### 과제 1. 로그인 흐름 추적

- UI 로그인 진입점 찾기
- Keycloak 관련 env 확인
- API auth 관련 엔드포인트 찾기
- Keycloak Provider 역할 확인

### 과제 2. 문서 조회 흐름 추적

- UI 문서 화면 찾기
- 호출 API 찾기
- API Controller / Service / Mapper 추적
- DB 쿼리 확인

### 과제 3. 파일 업로드 흐름 추적

- UI 업로드 시작 지점 찾기
- TUS 업로드 설정 확인
- API `/internal/tusd` 확인
- 후처리 로직 확인

## 11. 처음부터 깊게 보지 않아도 되는 파일

초기 탐색 단계에서는 아래는 나중으로 미뤄도 된다.

- 모든 Mapper XML 전체
- Kafka 세부 연동 코드
- RAG API 세부 구현
- MetaMask / QR 인증 상세 구현
- Helm 템플릿 전부

중요한 것은 모든 것을 읽는 것이 아니라, "핵심 흐름을 잃지 않는 것"이다.

## 12. 최종 한 줄

처음 읽을 때는 `UI 설정 -> API 진입점 -> 인증 확장 -> 업로드 서버 -> 운영 라우팅` 순서로 보면 가장 덜 헤맨다.
