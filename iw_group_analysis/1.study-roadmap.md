## 1. 먼저 잡아야 하는 큰 구조

이 그룹은 아래 5축으로 이해하면 된다.

1. 화면: `iop-iw-ui`
2. 업무 API: `iop-iw-api`
3. 인증: `keycloak`
4. 파일 업로드: `iop-iw-tus`
5. 배포/운영: `iop-iw-manifest`

보조 축은 아래 둘이다.

- `iop-iw-gpki`: GPKI 웹 로그인 보조 앱
- `user-storage-jpa-master`: Keycloak 확장 비교용 저장소

## 4. 추천 학습 순서

### 1단계: UI와 API 연결 이해

먼저 봐야 할 프로젝트:
- `iop-iw-ui`
- `iop-iw-api`

여기서 이해해야 할 것:
- 화면에서 어떤 API를 부르는지
- 토큰이 어떻게 붙는지
- 에디터/문서/첨부 기능이 어디로 가는지

이 단계가 가장 중요하다. 실제 업무 흐름 대부분이 여기서 시작된다.

### 2단계: 로그인 구조 이해

다음으로 봐야 할 프로젝트:
- `keycloak`
- 필요 시 `user-storage-jpa-master`

여기서 이해해야 할 것:
- UI가 Keycloak에 어떻게 로그인 요청을 보내는지
- API가 Keycloak 토큰을 어떻게 검증하는지
- 사용자 저장소를 Keycloak이 어떻게 외부 DB와 연결하는지

### 3단계: 파일 업로드 흐름 이해

다음으로 봐야 할 프로젝트:
- `iop-iw-tus`
- `iop-iw-api`

여기서 이해해야 할 것:
- UI가 업로드를 어디로 보내는지
- tusd가 업로드 완료 후 어떤 훅을 호출하는지
- API가 `/internal/tusd`에서 무엇을 후처리하는지

### 4단계: 운영/배포 구조 이해

다음으로 봐야 할 프로젝트:
- `iop-iw-manifest`

여기서 이해해야 할 것:
- `/iw`, `/iw/api`, `/iw/files`가 각각 어느 서비스로 가는지
- 환경별 값이 어떻게 갈리는지
- 실제 운영에서 어떤 서비스가 한 묶음으로 배포되는지

### 5단계: 특수 인증 흐름 이해

마지막으로 볼 프로젝트:
- `iop-iw-gpki`
- `keycloak`

여기서 이해해야 할 것:
- GPKI 로그인 화면/결과 처리가 별도 앱인지
- Keycloak의 `GpkiAuthenticator`와 어떻게 연결되는지

## 5. 프로젝트별 핵심 학습 포인트

## `iop-iw-ui`
- React 화면 구조
- API 호출 서비스 계층
- Keycloak 로그인 시작점
- 업로드 요청 시작점

## `iop-iw-api`
- Controller -> Service -> Mapper 흐름
- 인증 필터와 Security 설정
- 문서/첨부/관리자 기능 패키지 구조
- 다중 DataSource 구조

## `keycloak`
- Provider Factory 구조
- JDBC 사용자 저장소 구조
- Authenticator 실행 방식
- 세션 제한 이벤트 처리

## `iop-iw-tus`
- tusd 컨테이너 설정
- 업로드 경로와 저장 경로
- Hook 설정값

## `iop-iw-manifest`
- chart 구조
- environment values
- ingress / service 연결

## `iop-iw-gpki`
- JSP 진입 페이지
- GPKI 결과 처리 페이지
- WAR 배포 방식

## `user-storage-jpa-master`
- `keycloak`과 비교해 어떤 기능이 더 있는지
- 실험/이전 버전 흔적 확인

## 7. 한 줄 정리

이 그룹을 이해하는 가장 좋은 순서는 아래와 같다.

`UI 호출 흐름 -> API 업무 구조 -> Keycloak 인증 구조 -> TUS 업로드 흐름 -> Manifest 배포 구조 -> GPKI 특수 인증`
